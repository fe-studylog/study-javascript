Variables are containers for storing values.

When to use var, let, or const?

  1. Always use const if the value should not be changed.
  2. Always use const if the type should not be changed (Arrays and Objects).
  3. Only use let if you can't use const.
  4. Only use var if you must support old browsers.

JavaScript Identifiers are case-sensitive.

JavaScript Identifiers

  • Names can contain letters, digits, underscores, and dollar signs.
  • Names must begin width a letter.
  • Names are case sensitive (y and Y are different variables).
  • Reserved words can't be used as naems.

In JS, the equal sign(=) is an "assignment" operator, not an "equal to" operator.

The "equal to" operator is written like == in JS.

x = x + 5

  • It assigns the value of x + 5 to x.
  • It calculates the value of x + 5 and puts the result into x.

If you put a number in quotes, the rest of the numbers will be treated at strings, and concatenated.

JS Arithmetic

  • let x = "5" + 2 + 3 => x = 523
  • let y = 2 + 3 + "5" => x = 55

Variables defined with let can not be redeclared.

JavaScript Let

  • let x = "John Doe";
    let x = 0;
    => Not allowed
  • let x = 10; //  x = 10
    {
     let x = 2; //  x = 2
    }
    => Allowed

JavaScript const variables must be assigned a value when they are declared

Must be Assigned

  • const PI = 3.14159265359;
    => Correct
  • const PI;
    PI = 3.14159265359;
    => Incorrect

You can change the elements of a constant array. But you can NOT reassign the arrays.

Must be Assigned

  • const car = {type:"Fiat", model:"500", color:"white"};
    car.color = "red";
    => Correct
  • const car = {type:"Fiat", model:"500", color:"white"};
    car = {type:"Volvo", model:"EX60", color:"red"};
    => Error
